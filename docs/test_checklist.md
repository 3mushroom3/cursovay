# Чеклист тестирования (ручной) по ролям

## Гость
- Регистрация: ФИО + роль (студент/сотрудник) + документ (фото/PDF) + email/пароль.
- Авторизация.
- Сброс пароля.

## Пользователь (студент/сотрудник) со статусом `unverified`
- В ленте видит сообщение «ожидайте проверки профиля».

## Пользователь со статусом `disabled`
- В ленте видит сообщение об отказе.

## Пользователь со статусом `verified`
- Создание предложения: черновик / отправка на модерацию.
- Вложения: прикрепить фото/PDF, убедиться что URL появляется в деталях.
- Редактирование/удаление: доступно только в статусах `new`/`review`.
- Лента: видит свои + публичные чужие.
- Фильтры: статус + категория + поиск.
- Комментарии: автор может оставить комментарий в детали.
- Лайки: может лайкнуть публичное предложение.

## Модератор
- Видит все (публичные + свои + доступ через права).
- Меняет статус предложения (review → needs_info / at_work / completed / rejected).
- При `rejected` причина обязательна.
- После смены статуса появляется запись в истории.
- Добавляет комментарий в деталях.

## Администратор
- То же, что модератор.
- Админка пользователей: меняет роль и статус верификации, видит документ.
- Управление категориями: CRUD.
